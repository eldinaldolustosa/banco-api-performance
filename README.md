# banco-api-performance

## Introduction

This project contains performance tests for the Bank API, built with **K6** and **JavaScript**. The goal is to validate the application's behavior under load, checking response time, error rate, and stability of critical endpoints such as authentication and bank transfers.

---

## Technologies Used

| Technology | Version | Purpose |
|---|---|---|
| [K6](https://k6.io/) | Latest | Performance testing tool |
| JavaScript (ES6+) | — | Scripting language for tests |
| K6 Web Dashboard | K6 built-in | Real-time report visualization |


---

## Purpose of Each File Group

### `config/`
Stores the local project configuration. The `config.local.json` file defines the default `baseURL` used when no `BASE_URL` environment variable is provided at runtime.

### `fixtures/`
Contains static data (payloads) used in test requests. The `postLogin.json` file provides the authentication credentials sent to the `/login` endpoint.

### `helpers/`
Groups reusable support functions for tests. The `autenticacao.js` file exports the `obterToken()` function, which performs login and returns the JWT token — used by tests that require prior authentication, such as the transfers test.

### `reports/`
Output directory for HTML reports generated by the K6 Web Dashboard. Files generated here are listed in `.gitignore` and are not versioned.

### `tests/`
Contains the performance test scripts:
- **`login.test.js`**: Tests the `POST /login` endpoint with progressive load (stages), validating status 200 and JWT token response.
- **`transferencias.test.js`**: Tests the `POST /transferencias` endpoint with Bearer Token authentication, validating status 201 on transfer creation.

### `utils/`
Centralizes utility functions. The `variaveis.js` file exports `pegarBaseURL()`, which returns the `BASE_URL` environment variable (if set) or the value from `config.local.json` as a fallback.

## Installation and Execution

### Prerequisites

- [K6](https://k6.io/docs/get-started/installation/) installed on your machine

**Mac (Homebrew):**
```bash
brew install k6
```

**Linux (Debian/Ubuntu):**
```bash
sudo gpg -k
sudo gpg --no-default-keyring --keyring /usr/share/keyrings/k6-archive-keyring.gpg --keyserver hkp://keyserver.ubuntu.com:80 --recv-keys C5AD17C747E3415A3642D57D77C6C491D6AC1D69
echo "deb [signed-by=/usr/share/keyrings/k6-archive-keyring.gpg] https://dl.k6.io/deb stable main" | sudo tee /etc/apt/sources.list.d/k6.list
sudo apt-get update && sudo apt-get install k6
```

**Windows (Chocolatey):**
```bash
choco install k6
```

---

### Configuring baseURL

The `BASE_URL` variable can be passed directly via the command line or configured in the `config/config.local.json` file.

**Via environment variable (recommended):**
```bash
k6 run tests/login.test.js -e BASE_URL=http://localhost:3000
```

**Via configuration file (`config/config.local.json`):**
```json
{
  "baseURL": "http://localhost:3000"
}
```

---

### Running the Tests

**Login test:**
```bash
k6 run tests/login.test.js -e BASE_URL=http://localhost:3000
```

**Transfers test:**
```bash
k6 run tests/transferencias.test.js -e BASE_URL=http://localhost:3000
```

---

### Running with Real-Time Dashboard and HTML Export

The K6 Web Dashboard allows you to monitor metrics in real time in the browser and export the full report as an HTML file at the end of the run.

**Login test with dashboard and export:**
```bash
K6_WEB_DASHBOARD=true K6_WEB_DASHBOARD_EXPORT=./reports/login-html-report.html k6 run tests/login.test.js -e BASE_URL=http://localhost:3000
```

**Transfers test with dashboard and export:**
```bash
K6_WEB_DASHBOARD=true K6_WEB_DASHBOARD_EXPORT=./reports/transferencias-html-report.html k6 run tests/transferencias.test.js -e BASE_URL=http://localhost:3000
```

> After starting the command, access the real-time dashboard at: `http://127.0.0.1:5665`
>
> The exported HTML file will be available in the `reports/` folder once the run is complete.
